# 自动化求职助手系统 — 完整实施方案

---

## 第一步：三个项目分析

### 1.1 n8n — 自动化调度中枢

| 维度 | 详情 |
|------|------|
| **核心功能** | 工作流自动化平台，400+ 集成节点，支持定时触发、HTTP 请求、数据转换、条件分支等 |
| **角色定位** | **调度中枢 + 数据抓取引擎**：负责定时触发抓取任务、调用外部 API、串联各模块、记录日志 |
| **对外接口** | REST API（`/api/v1/`，Bearer Token 认证）、Webhook（`/webhook/`）、内部 API（`/rest/`） |
| **部署方式** | Docker（推荐）、npm（`npx n8n`） |
| **数据传递** | 工作流中通过 JSON 数据在节点间流转，支持 HTTP Request 节点调用外部服务 |

**关键能力**：
- **Schedule Trigger**：Cron 表达式定时触发（如每 6 小时一次）
- **HTTP Request**：调用 Boss 直聘搜索接口、调用 career-ops 评分 API
- **Code（JavaScript）**：在工作流中做数据清洗、JSON 解析
- **Webhook**：接收 geekgeekrun 的投递结果回调
- **Postgres/SQLite 节点**：直接读写数据库
- **IF/Switch**：条件判断（分数 > 80 则投递，否则跳过）

### 1.2 career-ops — AI 评分引擎

| 维度 | 详情 |
|------|------|
| **核心功能** | AI 驱动的职位评估管线：扫描门户 → 过滤去重 → AI 评估打分 → 生成简历 PDF → 追踪记录 |
| **角色定位** | **智能筛选 + 评分模块**：接收 JD 文本，输出匹配度分数（0-100 或 1-5 分制），提供详细评估报告 |
| **对外接口** | 命令行（`node gemini-eval.mjs "JD文本"`）、`node scan.mjs` 扫描门户（无 REST API，需封装） |
| **部署方式** | Node.js（`npm install` + 环境变量配置） |
| **数据传递** | 输入：JD 文本或文件；输出：控制台文本 + `reports/` 目录下的 Markdown 报告 + `data/applications.md` 追踪器 |

**关键脚本**：
- **`scan.mjs`**：零 Token 消耗的门户扫描器，支持 Greenhouse、Ashby、Lever 等 ATS 平台
- **`gemini-eval.mjs`**：使用 Gemini API 对 JD 进行 A-G 七模块评估，输出 1-5 分评分
- **评分体系**：4.5+ = 强匹配，4.0-4.4 = 好匹配，3.5-3.9 = 一般，<3.5 = 不建议

**局限性**：
- 原生不提供 REST API，需要封装为 HTTP 服务供 n8n 调用
- 主要面向海外 ATS 平台（Greenhouse/Ashby/Lever），不直接支持 Boss 直聘等国内平台
- 评分 prompt 需要根据嵌入式/Linux 开发岗位进行定制

### 1.3 geekgeekrun — Boss 直聘自动投递

| 维度 | 详情 |
|------|------|
| **核心功能** | 基于 Electron + Puppeteer 的 Boss 直聘自动批量开聊工具，支持自动筛选、标记不合适、已读不回复聊 |
| **角色定位** | **自动投递模块**：在 Boss 直聘上自动与匹配的 Boss 开聊（即"投递"） |
| **对外接口** | Electron CLI 模式（`--mode=geekAutoStartWithBoss`）、出站 Webhook（将结果发送到外部）、无入站 REST API |
| **部署方式** | Electron 桌面应用（Windows/macOS/Linux），需 GUI 环境（Puppeteer 默认 `headless: false`） |
| **数据传递** | 配置文件（`~/.geekgeekrun/config/boss.json`）定义筛选条件；SQLite（`~/.geekgeekrun/storage/public.db`）存储记录 |

**关键特性**：
- **自动开聊**：根据配置的职位来源、筛选条件自动与 Boss 沟通
- **多维度筛选**：城市、薪资、经验、职位名称/类型/描述正则、BOSS 活跃度
- **反检测**：StealthPlugin + LaodengPlugin + UA 匿名化
- **已读不回复聊**：自动跟进已读不回的 Boss
- **出站 Webhook**：可将开聊结果 POST 到指定 URL

**局限性**：
- **无 headless 模式**：Puppeteer 硬编码为 `headless: false`，需要 GUI 环境（不能直接在纯服务器上运行）
- **无入站 REST API**：不能通过 HTTP 触发，只能通过 Electron CLI 或直接 `node` 启动
- **需手动登录**：首次使用需通过"BOSS 登录助手"扫码登录，Cookie 保存在本地
- **平台风控**：高频操作可能触发 Boss 直聘风控，需要"贤者时间"（自动暂停）机制

### 1.4 三者协作关系

```
┌─────────────────────────────────────────────────────────────────────┐
│                        n8n（调度中枢）                               │
│  · 定时触发（每 6 小时）                                              │
│  · 数据清洗 & 去重                                                   │
│  · 条件判断（分数 > 80）                                              │
│  · 调用外部服务                                                      │
│  · 记录日志到数据库                                                   │
└────────┬────────────────────────┬───────────────────────────────────┘
         │                        │
         ▼                        ▼
┌─────────────────┐    ┌──────────────────────────────────┐
│  career-ops      │    │  geekgeekrun                     │
│  （AI 评分引擎）  │    │  （Boss 直聘自动投递）             │
│                  │    │                                   │
│  · 接收 JD 文本  │    │  · 接收高匹配岗位列表               │
│  · AI 匹配评分   │    │  · 自动与 Boss 开聊                │
│  · 输出评估报告  │    │  · 记录投递结果                    │
└─────────────────┘    └──────────────────────────────────┘
```

**数据传递方式**：
1. **n8n → career-ops**：n8n 通过 HTTP Request 调用 career-ops 的评分 API（需封装为 Flask/FastAPI 服务）
2. **n8n → geekgeekrun**：方案 A：n8n 通过 SSH + CLI 命令触发 geekgeekrun；方案 B：n8n 将高匹配岗位信息通过 Webhook 推送给 geekgeekrun
3. **career-ops → n8n**：评分结果通过 HTTP 响应返回给 n8n
4. **geekgeekrun → n8n**：投递结果通过 geekgeekrun 的出站 Webhook 回传给 n8n

### 1.5 当前已实现方案：GitHub Actions + DeepSeek API

> **实际落地版本**（2026-05 更新）：为降低部署复杂度，采用 GitHub Actions 替代 n8n 做定时调度，用 DeepSeek API 替代 career-ops 做 AI 评分。

| 维度 | 原规划 | 当前实现 |
|------|--------|----------|
| **调度平台** | n8n (Docker) | GitHub Actions (免费 CI/CD) |
| **AI 评分** | career-ops + Gemini API | DeepSeek API（直接 HTTP 调用，不再依赖 career-ops） |
| **抓取方式** | n8n HTTP Request 节点 | Python 脚本（DrissionPage + curl_cffi） |
| **数据库** | PostgreSQL | JSON 文件（`data/seen_jobs.json`），服务器端可选 SQLite/PostgreSQL |
| **通知** | n8n 钉钉节点 | Python 直接调用钉钉 Webhook（支持 ActionCard 反馈按钮） |
| **部署成本** | 需要阿里云服务器 | 零成本（GitHub 免费额度）；服务器部署为可选 |
| **评分服务** | Flask 封装 career-ops 脚本 | Flask 直接调用 DeepSeek API（`deploy/scoring-service/`） |

`deploy/scoring-service/app.py` 核心逻辑：

```python
import json
import os
import requests
from flask import Flask, request, jsonify

app = Flask(__name__)

MIMO_API_KEY = os.environ.get('MIMO_API_KEY', '')
MIMO_BASE_URL = os.environ.get('MIMO_BASE_URL', 'https://token-plan-cn.xiaomimimo.com/v1')
MIMO_MODEL = os.environ.get('MIMO_MODEL', 'mimo-v2.5-pro')


def evaluate_with_mimo(jd_text):
    if not MIMO_API_KEY:
        return {"error": "MIMO_API_KEY 未设置", "success": False}

    url = f"{MIMO_BASE_URL}/chat/completions"
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {MIMO_API_KEY}",
    }

    prompt = f"""你是一个求职匹配评估专家。请根据以下简历信息和职位描述，给出0-100的匹配度评分。

## 我的背景
- 大三本科生，长春大学旅游学院，物联网工程专业
- 求职意向：嵌入式软件/Linux应用开发实习生
- 熟练掌握C，熟悉C++面向对象编程，了解Python
- 熟悉Linux系统编程（进程、线程、文件I/O、Socket），掌握TCP/UDP协议及epoll高并发模型
- 了解嵌入式Linux开发流程，掌握UART、I2C、SPI等通信协议
- 项目经验：基于MQTT的智能家居控制系统（i.MX6ULL开发板，MQTT，QT，InfluxDB，Grafana，Docker）
- 英语六级（CET-6），国家励志奖学金

## 评分标准（注意：我是找实习的在校生，不是社招）
- 90-100: 完美匹配，必须投递（嵌入式/Linux开发实习，技术栈高度匹配）
- 80-89: 高度匹配，建议投递（嵌入式/Linux相关实习，大部分技能匹配）
- 70-79: 一般匹配，可考虑（相关领域实习，部分技能可迁移）
- 60以下: 不太匹配（方向不相关，如纯前端、纯Java后端等）

## 职位描述
{jd_text}

请只返回一个JSON格式：{{"score": 分数, "reason": "简短理由"}}
"""

    payload = {
        "model": MIMO_MODEL,
        "messages": [
            {"role": "system", "content": "你是一个专业的求职匹配评估专家，只返回JSON格式的评分结果。"},
            {"role": "user", "content": prompt}
        ],
        "temperature": 0.1,
        "max_tokens": 300,
    }

    resp = requests.post(url, headers=headers, json=payload, timeout=30)
    result = resp.json()
    # 解析 MiMo 响应中的 choices/content 字段
    text = result["choices"][0]["message"]["content"]
    text = text.strip().replace("```json", "").replace("```", "").strip()
    parsed = json.loads(text)
    return {"score": parsed.get("score", 0), "reason": parsed.get("reason", ""), "success": True}


@app.route('/health', methods=['GET'])
def health():
    return jsonify({'status': 'ok'})


@app.route('/evaluate', methods=['POST'])
def evaluate():
    data = request.get_json(force=True, silent=True)
    if not data or 'jd' not in data:
        return jsonify({'error': 'Missing "jd" field'}), 400

    jd_text = data['jd']
    company = data.get('company', 'Unknown')
    role = data.get('role', 'Unknown')

    result = evaluate_with_mimo(jd_text)
    if not result.get('success'):
        return jsonify({'company': company, 'role': role, 'score': 0, 'error': result.get('error'), 'success': False})

    return jsonify({'company': company, 'role': role, 'score': result['score'], 'reason': result['reason'], 'success': True})


@app.route('/evaluate-batch', methods=['POST'])
def evaluate_batch():
    data = request.get_json(force=True, silent=True)
    if not data or 'jobs' not in data:
        return jsonify({'error': 'Missing "jobs" field'}), 400

    results = []
    for job in data['jobs'][:10]:
        jd = job.get('jd', '')
        if not jd:
            results.append({'error': 'Empty JD', 'success': False})
            continue
        result = evaluate_with_mimo(jd)
        results.append({
            'company': job.get('company', 'Unknown'),
            'role': job.get('role', 'Unknown'),
            'score': result.get('score', 0),
            'reason': result.get('reason', ''),
            'success': result.get('success', False)
        })

    return jsonify({'results': results})


if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=False)
```

### 4.2 Python 评分脚本（独立调用）

当前版本 `deploy/scoring-service/score_job.py` 直接调用 MiMo API：

```python
#!/usr/bin/env python3
import sys
import json
import os
import requests

MIMO_API_KEY = os.environ.get('MIMO_API_KEY', '')
MIMO_BASE_URL = os.environ.get('MIMO_BASE_URL', 'https://api.xiaomimimo.com/v1')
MIMO_MODEL = os.environ.get('MIMO_MODEL', 'mimo-v2.5-pro')


def extract_mimo_response_text(result):
    if "choices" in result:
        try:
            return result["choices"][0]["message"]["content"]
        except (KeyError, IndexError, TypeError):
            pass
    for field in ["result", "response", "output", "text", "content"]:
        val = result.get(field)
        if isinstance(val, str) and val.strip():
            return val.strip()
    data = result.get("data")
    if isinstance(data, dict):
        if "choices" in data:
            try:
                return data["choices"][0]["message"]["content"]
            except (KeyError, IndexError, TypeError):
                pass
        for field in ["result", "response", "output", "text", "content"]:
            val = data.get(field)
            if isinstance(val, str) and val.strip():
                return val.strip()
    return None


def evaluate_job(jd_text, company="Unknown", role="Unknown"):
    if not MIMO_API_KEY:
        return {"error": "MIMO_API_KEY 未设置", "success": False}

    url = f"{MIMO_BASE_URL}/chat/completions"
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {MIMO_API_KEY}",
    }

    prompt = f"""你是一个求职匹配评估专家。请根据以下简历信息和职位描述，给出0-100的匹配度评分。

## 我的背景
- 大三本科生，长春大学旅游学院，物联网工程专业
- 求职意向：嵌入式软件/Linux应用开发实习生
- 熟练掌握C，熟悉C++面向对象编程，了解Python
- 熟悉Linux系统编程（进程、线程、文件I/O、Socket），掌握TCP/UDP协议及epoll高并发模型
- 了解嵌入式Linux开发流程，掌握UART、I2C、SPI等通信协议
- 项目经验：基于MQTT的智能家居控制系统（i.MX6ULL开发板，MQTT，QT，InfluxDB，Grafana，Docker）
- 英语六级（CET-6），国家励志奖学金

## 评分标准（注意：我是找实习的在校生，不是社招）
- 90-100: 完美匹配，必须投递（嵌入式/Linux开发实习，技术栈高度匹配）
- 80-89: 高度匹配，建议投递（嵌入式/Linux相关实习，大部分技能匹配）
- 70-79: 一般匹配，可考虑（相关领域实习，部分技能可迁移）
- 60以下: 不太匹配（方向不相关，如纯前端、纯Java后端等）

## 职位描述
{jd_text}

请只返回一个JSON格式：{{"score": 分数, "reason": "简短理由"}}
"""

    payload = {
        "model": MIMO_MODEL,
        "messages": [
            {"role": "system", "content": "你是一个专业的求职匹配评估专家，只返回JSON格式的评分结果。"},
            {"role": "user", "content": prompt}
        ],
        "temperature": 0.1,
        "max_tokens": 300,
    }

    try:
        resp = requests.post(url, headers=headers, json=payload, timeout=30)
        result = resp.json()
        text = extract_mimo_response_text(result)
        if not text:
            return {"error": "无法解析MiMo响应", "success": False}
        text = text.strip().replace("```json", "").replace("```", "").strip()
        parsed = json.loads(text)
        return {
            "company": company,
            "role": role,
            "score": parsed.get("score", 0),
            "reason": parsed.get("reason", ""),
            "success": True
        }
    except Exception as e:
        return {"error": str(e), "success": False}


if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("用法: python3 score_job.py <jd文本> [公司名] [职位名]")
        sys.exit(1)

    jd = sys.argv[1]
    company = sys.argv[2] if len(sys.argv) > 2 else "Unknown"
    role = sys.argv[3] if len(sys.argv) > 3 else "Unknown"

    result = evaluate_job(jd, company, role)
    print(json.dumps(result, ensure_ascii=False, indent=2))
```

> **注意**：评分 prompt 中的"我的背景"需要根据实际简历内容修改，以获得更准确的匹配评分。

| **实习僧** | ⭐⭐ | JS 渲染 | DrissionPage |

#### Boss 直聘常见错误码

| code | 含义 | 原因 | 解决方案 |
|------|------|------|----------|
| `0` | 正常 | - | - |
| `1` | 业务错误 | Cookie 过期/无效 | 重新获取 Cookie |
| `36` | 账户异常 | 账号被标记为异常行为 | 等待 24-48 小时自动解封，期间不要用任何自动化工具 |
| `403` | Forbidden | IP 被封或 TLS 指纹被识别 | 使用 curl_cffi 或 DrissionPage |
| `429` | 限流 | 请求频率过高 | 增大请求间隔（建议 8-18 秒） |

#### 反爬检测维度

| 检测维度 | 说明 | 当前应对方案 |
|----------|------|-------------|
| **TLS 指纹** | Python `requests` 的 TLS 指纹与 Chrome 不同 | 使用 `curl_cffi` 的 `impersonate="chrome131"` 模拟 |
| **请求头完整性** | 缺少 `Sec-Ch-Ua`、`Sec-Fetch-*` 等头 | 添加完整的 Chrome 浏览器请求头 |
| **请求频率** | 固定间隔的请求会被识别 | 随机化延迟（8-18 秒）+ 指数退避重试 |
| **IP 信誉** | GitHub Actions 的云端 IP 被标记 | 使用 DrissionPage 真实浏览器（TLS 指纹来自真实 Chrome） |
| **JavaScript 验证** | 页面需要执行 JS 才能获取数据 | DrissionPage 运行真实 Chrome，自动执行 JS |
| **账户行为** | 同一账户频繁被用于自动化 | 降低频率，不要 24 小时不停运行 |
| **WAF 防护** | 阿里云 WAF 拦截非浏览器请求 | DrissionPage 绕过 WAF |

#### DrissionPage 浏览器自动化（主力方案）

DrissionPage 使用真实 Chrome 浏览器，能绕过大部分反爬检测：
- TLS 指纹来自真实 Chrome
- 自动执行 JavaScript

- 支持 headless 模式（无 GUI 环境）
- 自动处理安全验证页面

### H. 投递反馈功能说明（2026-05-21 新增）

#### 功能概述

系统现在支持记录和跟踪投递状态，帮助你分析求职效果。

#### 数据存储

反馈记录保存在 `data/feedback_records.json`，结构如下：

```json
{
  "job_id_1": {
    "company": "华为技术",
    "role": "嵌入式Linux开发实习",
    "score": 85,
    "salary": "200-300元/天",
    "location": "深圳",
    "url": "https://www.zhipin.com/job_detail/xxx.html",
    "action": "applied",
    "notified_at": "2026-05-21T08:00:00",
    "updated_at": "2026-05-21T10:30:00"
  }
}
```

#### 状态说明

| 状态 | 含义 | 触发方式 |
|------|------|----------|
| `notified` | 已通知（默认） | 自动：发送钉钉通知时 |
| `applied` | 已投递 | 手动：点击按钮或运行命令 |
| `rejected` | 不合适 | 手动：点击按钮或运行命令 |
| `viewed` | 已查看 | 手动：点击按钮或运行命令 |

#### 使用方法

**方法一：钉钉通知按钮**

高分岗位（≥80分）的钉钉通知会显示三个按钮：
- ✅ 已投递
- ❌ 不合适
- 👀 已查看

**方法二：命令行工具**

```bash
# 查看所有记录
python update_feedback.py list

# 查看已投递的记录
python update_feedback.py list applied

# 更新状态为"已投递"
python update_feedback.py update <job_id> applied

# 查看统计
python update_feedback.py stats
```

#### 统计报告

每日抓取报告会自动包含反馈统计：

```
📈 投递反馈统计（累计）
- 总通知: 50 | 已投递: 15 | 不合适: 10 | 待处理: 25
- 投递率: 30.0%
```

#### 分析建议

1. **投递率**：30-50% 为正常范围
2. **拒绝率**：如果过高，考虑调整 `profile.json` 中的黑名单关键词
3. **待处理**：定期处理，避免错过好岗位
